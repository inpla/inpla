# Visualising Parallel Rewriting with `parallel_analysis.plt`

This document explains how to use the gnuplot script `parallel_analysis.plt` to visualise execution
profiles produced by Inpla in parallel execution simulation mode.

The script plots:

- the number of **executed active pairs per step** (stacked bars, left y-axis), and
- the **number of agents** present in the net (line plot, right y-axis),

based on CSV output generated by Inpla.



## 1. Prerequisites

You will need:

- **Inpla** (parallel execution simulation mode enabled)
- **gnuplot** (with `cairo` terminals recommended)
- CSV data generated by Inpla using the `-s` option

The expected CSV header is of the form:

```text
step,exec(<num>),stacked,agents,rule1,rule2,...
```

The first row is assumed to be a header and is skipped automatically by the script.



## 2. Recommended Directory Layout

```text
project/
├── inpla
├── add1.in
├── add2.in
├── tmp.csv
└── tools/
    ├── parallel_analysis.plt
    └── parallel_analysis.md
```

All example commands below assume execution from the project root directory.


## 3. Basic Usage

If your CSV file is named `tmp.csv`, simply run:

```bash
gnuplot tools/parallel_analysis.plt
```

This generates a PDF file named `analysis.pdf` using default settings.



## 4. Typical Workflow with Inpla

### Step 1: Run Inpla in parallel execution simulation mode

CSV output is written to `STDERR`, so redirect it to a file:

```bash
./inpla -f add1.in -s 0 2> tmp.csv
```

Here, `-s 0` enables parallel execution simulation with unbounded parallelism.

### Step 2: Visualise the execution profile

```bash
gnuplot -e "file='tmp.csv'; outfile='add1.eps'; title_text='(a+b)+c'" tools/parallel_analysis.plt
```



## 5. Output Formats

The output format is selected automatically from the extension of `outfile`:

- `.pdf` (default)
- `.eps`
- `.png`

Example:

```bash
gnuplot -e "file='tmp.csv'; outfile='result.png'" tools/parallel_analysis.plt
```



## 6. Figure Size and Font Control (All Formats)

The script uses format-independent parameters for figure size and font control.

### Parameters

| Parameter   | Description                     | Default |
|------------|---------------------------------|---------|
| `fig_w`    | figure width (inches)           | 3.5     |
| `fig_h`    | figure height (inches)          | 2.5     |
| `font_size`| base font size                  | 12      |
| `png_dpi`  | DPI for PNG output              | 200     |

These parameters apply uniformly to EPS, PDF, and PNG outputs.
For PNG output, inches are converted internally using `png_dpi`.

Example:

```bash
gnuplot -e "file='tmp.csv'; outfile='fig.eps'; fig_w=4.8; fig_h=3.2; font_size=10" tools/parallel_analysis.plt
```



## 7. Legend (Key) Control

The legend (key) can be shown or hidden and its font size adjusted.

### Parameters

| Parameter    | Description                                  | Default |
|--------------|----------------------------------------------|---------|
| `show_key`   | show or hide legend (`1` = show, `0` = hide) | 1       |
| `key_fsize`  | legend font size                             | follows `font_size` |

### Examples

Hide the legend (useful for side-by-side figures):

```bash
gnuplot -e "file='tmp.csv'; outfile='fig.eps'; show_key=0" tools/parallel_analysis.plt
```

Make the legend text smaller:

```bash
gnuplot -e "file='tmp.csv'; outfile='fig.eps'; key_fsize=8" tools/parallel_analysis.plt
```



## 8. Axis Limits and Tick Control

To facilitate precise comparison between multiple runs, axis limits and tick intervals can be fixed.

### Parameters

| Parameter | Description |
|----------|-------------|
| `x_max`  | maximum step shown |
| `y_max`  | maximum executions (left axis) |
| `y2_max` | maximum agents (right axis) |
| `x_step` | x-axis tick interval |
| `y_step` | y-axis tick interval |
| `y2_step`| y2-axis tick interval |

Example:

```bash
gnuplot -e "file='tmp.csv'; x_max=150; y_max=3000; y_step=500; y2_max=200; y2_step=50" tools/parallel_analysis.plt
```



## 9. Plot Contents

The generated figure contains:

- **Stacked bar charts** (left y-axis): number of active pairs executed per step, broken down by rewrite rules.
- **Line plot** (right y-axis): total number of agents present at each step.

This representation makes it possible to observe both execution activity and structural growth of the net during evaluation.



## 10. Intended Use

The script is intended as a lightweight visualisation tool to:

- analyse parallel rewriting behaviour in interaction nets,
- compare different formulations of the same computation,
- study the effect of bounded vs unbounded parallelism,
- support experimental results in research papers and system descriptions.

In particular, it is well suited for illustrating how the available degree of parallelism influences observed performance.



## 11. Summary

The basic workflow is:

1. run Inpla and redirect `STDERR` to a CSV file,
2. visualise the CSV data using `parallel_analysis.plt`,
3. adjust figure size, fonts, and legend options as needed.

Minimal example:

```bash
./inpla -f add1.in -s 0 2> tmp.csv
gnuplot -e "file='tmp.csv'; outfile='add1.pdf'" tools/parallel_analysis.plt
```

This provides a simple and reproducible workflow for analysing the dynamics of parallel rewriting.
